Java I/O、NIO 和 Okio

I/O 是什么

       程序内部和外部进行数据交互的过程，就叫输入输出。程序内部指的是内存，比如程序中的 String、int；程序外部指的是内存之外的部分，一般指的是本地文件（磁盘）和网络。也有别的情况，比如你和别的程序做交互，和你交互的程序也属于外部，但一般
来说，就是文件和网络这两种目标。从文件里或者从网络上读数据到内存里，就叫输入；从内存里写到文件里或者发送到网络上，就叫输出。Java I/O 作用只有一个——和外界做数据交互。

NIO

       NIO 和 IO 的区别有几点：

  1. 传统 IO（Java I/O） 通过插管道的方式来读写，用的是 Stream； NIO 也是插管道的方式，用的是 Channel。传统 IO 是单向的，要么读要么写，NIO 的 Channel 是双向的；
  2. NIO 也用到 buffer，它的 Buffer 可以被精细操作，它强制使用 Buffer，但是它的 buffer 不好用；
  3. NIO 有非阻塞式的支持，只是支持非阻塞式，而不是全是非阻塞式，默认是阻塞式的。
    就算是非阻塞式，也只是支持网络交互，文件交互是不支持的。
